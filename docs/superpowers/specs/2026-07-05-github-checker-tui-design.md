# github-checker — TUI-монитор GitHub-репозиториев

**Дата:** 2026-07-05
**Статус:** утверждено

## Цель

Утилита с TUI-дашбордом, которая отслеживает состояние набора GitHub-репозиториев
(состав меняется) и показывает: открытые PRы, ветки, активность dependabot
(PRы + security alerts) и статус ревью от GitHub Copilot.

## Решения, принятые при обсуждении

- Источник: удалённые репозитории GitHub по списку `owner/repo` (не локальные клоны).
- Доступ к API: `gh` CLI (`gh api ...`) через `asyncio.subprocess` —
  авторизация уже настроена у пользователя, токенами не управляем.
- Dependabot: и PRы от `dependabot[bot]` (помечаются в списке PR),
  и счётчик открытых security alerts.
- Copilot: для каждого PR — есть ли ревью от `copilot-pull-request-reviewer[bot]`,
  его статус и число комментариев. Тексты комментариев не показываем.
- Режим: живой интерактивный дашборд на Textual с автообновлением.
- Конфиг: TOML-файл + добавление/удаление реп горячими клавишами из TUI
  с записью обратно в файл.

## Конфигурация

Файл `repos.toml` в корне проекта; путь переопределяется флагом `--config`.

```toml
repos = [
    "andrei-shtanakov/atp-platform",
    "andrei-shtanakov/Maestro",
    "andrei-shtanakov/spec-runner",
]
refresh_seconds = 120
```

- Чтение: `tomllib` → pydantic-модель `Config` (валидация формата `owner/repo`).
- Запись (после add/remove из TUI): `tomli_w`. Файл переписывается целиком,
  комментарии в нём не сохраняются — это осознанное упрощение.
- Если файла нет — создаётся пустой шаблон и TUI показывает подсказку
  «добавьте репозиторий клавишей a».

## Архитектура

~5 небольших модулей в пакете `github_checker/`:

| Модуль | Ответственность |
|---|---|
| `models.py` | pydantic-модели: `Config`, `PullRequest`, `CopilotReview`, `Branch`, `RepoState` |
| `config.py` | загрузка/сохранение `repos.toml` |
| `github.py` | асинхронные вызовы `gh api`, маппинг JSON → модели |
| `app.py` | Textual-приложение: виджеты, клавиши, таймер обновления |
| `main.py` | entrypoint: разбор `--config`, запуск приложения |

Поток данных: таймер/клавиша → `github.fetch_all(config)` →
`list[RepoState]` → обновление виджетов. Слой данных не знает о TUI,
TUI не знает о subprocess — граница проходит по `list[RepoState]`.

## Слой данных (`github.py`)

На каждую репу параллельно (`asyncio.gather`, ограничение семафором ~8
одновременных процессов `gh`):

1. `gh api repos/{r}/pulls?state=open&per_page=100` —
   номер, заголовок, автор, head-ветка, `is_dependabot` (автор `dependabot[bot]`).
2. Для каждого открытого PR: `gh api repos/{r}/pulls/{n}/reviews` —
   ревью от `copilot-pull-request-reviewer[bot]`: статус (`APPROVED` /
   `CHANGES_REQUESTED` / `COMMENTED`) и число ревью-комментариев.
3. `gh api repos/{r}/branches?per_page=100` — имена веток.
4. `gh api repos/{r}/dependabot/alerts?state=open&per_page=100` —
   счётчик открытых уязвимостей.

Обработка ошибок:

- Нет прав на dependabot alerts (403/404) → в `RepoState.alerts = None`,
  в таблице «n/a». Не считается ошибкой репы.
- Любая другая ошибка по репе (нет доступа, репа удалена, сеть) →
  `RepoState.error: str`, строка репы показывает статус ошибки;
  остальные репы обновляются как обычно.
- `gh` не установлен / не авторизован → человекочитаемое сообщение при старте,
  выход с кодом 1.
- Пагинация: одна страница `per_page=100`; если элементов ровно 100,
  в ячейке показывается `100+`. Дальнейшие страницы не тянем (YAGNI).

## TUI (`app.py`, Textual)

Раскладка: две панели.

- **Слева — таблица реп** (DataTable), колонки:
  `Repo | PRs | Bot | Branches | Alerts | Copilot | Updated`
  - `Bot` — сколько из открытых PR от dependabot;
  - `Copilot` — сводка по репе: `n/m` (n PR с ревью Copilot из m открытых);
  - `Updated` — время последнего успешного обновления (HH:MM:SS);
  - строка с ошибкой подсвечивается, в `Updated` — `error`.
- **Справа — детали выбранной репы**:
  - список PR: `#номер заголовок (автор)` с бейджами `[dbot]` для dependabot
    и `[copilot: approved|changes|commented (k)]` при наличии ревью;
  - список веток.

Клавиши:

| Клавиша | Действие |
|---|---|
| `r` | обновить все репы сейчас (конфиг перечитывается с диска) |
| `a` | модальный ввод `owner/repo` → валидация → запись в TOML → загрузка |
| `d` | удалить выбранную репу (с подтверждением) → запись в TOML |
| `q` | выход |
| `↑/↓` | навигация по таблице, детали обновляются |

Автообновление по таймеру `refresh_seconds` (Textual `set_interval`).
Во время обновления — индикатор в футере; интерфейс не блокируется
(fetch выполняется в фоне через `run_worker`).

## Зависимости

- `textual` — TUI
- `pydantic` — модели и валидация
- `tomli-w` — запись конфига (чтение — stdlib `tomllib`)
- dev: `pytest`, `anyio`, `pyrefly`, `ruff`

Внешнее требование: установленный и авторизованный `gh` CLI.

## Тестирование

- `pytest` + `anyio` (async-тесты без asyncio-маркеров).
- `config.py`: чтение/запись TOML, невалидные строки `owner/repo`,
  отсутствующий файл, roundtrip add/remove.
- `github.py`: маппинг JSON → модели на фикстурах с реальными ответами API
  (сохранённых в `tests/fixtures/`); пометка dependabot; выделение
  copilot-ревью; ветки 403/404 для alerts; ошибка репы.
  Subprocess мокается — сетевых вызовов в тестах нет.
- `app.py`: минимально через `App.run_test()` — рендер таблицы из готовых
  `RepoState`, обработка клавиши `d` с подтверждением.

## Вне рамок (YAGNI)

- Тексты ревью-комментариев Copilot.
- Действия над PR (merge, close) из TUI.
- Полная пагинация свыше 100 элементов.
- Уведомления, звук, история изменений.
- Поддержка GitHub Enterprise / нескольких аккаунтов.
